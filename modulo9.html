Módulo 9: Copias de Seguridad y Restauración

1. Tipos de Copias de Seguridad

SQL Server ofrece varios tipos de copias de seguridad para proteger los datos y minimizar la pérdida en caso de fallos:

  • Copia completa: Realiza un respaldo completo de toda la base de datos. Es la base para cualquier estrategia de recuperación.
  • Copia diferencial: Almacena solo los cambios realizados desde la última copia completa, ocupando menos espacio y tiempo.
  • Copia del registro de transacciones: Registra todas las transacciones desde la última copia del registro, permitiendo una recuperación granular.

Ejemplo práctico: Realizar una copia completa


    -- Crear una copia completa de la base de datos
    BACKUP DATABASE MiBaseDeDatos 
    TO DISK = 'C:\Respaldos\MiBaseDeDatos_Completa.bak'
    WITH FORMAT, MEDIANAME = 'Media1', NAME = 'CopiaCompleta';

2. Modelos de Recuperación

Los modelos de recuperación determinan cómo se almacenan los datos en el registro de transacciones y afectan las opciones de restauración:

  • Modelo Simple: No guarda el historial de transacciones, por lo que es ideal para bases de datos no críticas. No permite restauración a un punto en el tiempo.
  • Modelo Completo: Registra todas las transacciones, permitiendo restauraciones detalladas. Requiere un manejo regular de los registros.
  • Modelo de Registro Masivo: Optimizado para operaciones masivas, como cargas de datos. Consume menos espacio pero limita la recuperación detallada.

Ejemplo práctico: Configurar el modelo de recuperación


    -- Cambiar el modelo de recuperación a Completo
    ALTER DATABASE MiBaseDeDatos 
    SET RECOVERY FULL;

3. Uso Práctico: Copias y Restauraciones

SQL Server Management Studio (SSMS) proporciona una interfaz gráfica para realizar copias de seguridad y restauraciones fácilmente. También se puede usar Transact-SQL para mayor control.

Ejemplo práctico: Restaurar una copia completa


    -- Restaurar una copia completa de la base de datos
    RESTORE DATABASE MiBaseDeDatos 
    FROM DISK = 'C:\Respaldos\MiBaseDeDatos_Completa.bak'
    WITH REPLACE, 
    MOVE 'MiBaseDeDatos_Data' TO 'C:\BasesDeDatos\MiBaseDeDatos.mdf',
    MOVE 'MiBaseDeDatos_Log' TO 'C:\BasesDeDatos\MiBaseDeDatos.ldf';

4. Estrategias de Respaldo

Es crucial definir una estrategia basada en las necesidades de tu organización:

  • Realizar copias completas semanales.
  • Hacer copias diferenciales diarias para minimizar los tiempos de respaldo.
  • Configurar copias del registro de transacciones cada hora para escenarios críticos.

Consejo: Almacena copias en ubicaciones externas o en la nube para proteger los datos ante desastres físicos.

5. Recuperación a un Punto en el Tiempo

Esta técnica permite restaurar la base de datos a un momento exacto utilizando las copias del registro de transacciones. Es ideal para revertir errores accidentales.

Ejemplo práctico: Restaurar a un punto en el tiempo


    -- Restaurar la base de datos a un punto específico
    RESTORE DATABASE MiBaseDeDatos 
    FROM DISK = 'C:\Respaldos\MiBaseDeDatos_Completa.bak' 
    WITH NORECOVERY;
    
    RESTORE LOG MiBaseDeDatos 
    FROM DISK = 'C:\Respaldos\MiBaseDeDatos_Log.trn' 
    WITH STOPAT = '2024-12-01T10:15:00', RECOVERY;

6. Mejores Prácticas

  • Automatiza las copias con el Agente de SQL Server para garantizar consistencia.
  • Verifica regularmente la integridad de las copias usando el comando RESTORE VERIFYONLY.
  • Documenta los procedimientos de recuperación y realiza simulacros periódicos para validar la estrategia.

Conclusión

Las copias de seguridad y restauración son fundamentales para proteger los datos y asegurar la continuidad operativa. Implementar una estrategia robusta y entender las opciones disponibles permite gestionar riesgos y minimizar tiempos de inactividad.
